# DESMET Agentic Platforms - Pipeline Stages

This document describes the software engineering pipeline for evaluating
agentic platforms using representative tasks.

## Pipeline Overview

```
┌─────────────────────┐
│  1. REQUIREMENTS    │  ◄── Natural language requirements
│     STAGE           │  ──► User stories, PlantUML diagrams, API specs
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  2. CODE GENERATION │  ◄── Entities, APIs, components from Stage 1
│     STAGE           │  ──► Generated code, project structure
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  3. TESTING         │  ◄── Acceptance criteria, code from Stages 1-2
│     STAGE           │  ──► Unit tests, integration tests, test reports
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  4. BUILD & DEPLOY  │  ◄── Code, tests, configs from Stages 1-3
│     STAGE           │  ──► Built artifacts, deployment configs
└─────────────────────┘
```

## Stage 1: Requirements Stage (IMPLEMENTED)

**Location:** `stages/requirements/`

**Input:**
- Natural language requirements
- User stories
- Feature requests
- Stakeholder information
- Constraints

**Output:**
- Structured user stories with acceptance criteria
- Functional requirements
- Non-functional requirements
- Use case specifications
- Domain entity/class designs
- System component architecture
- API endpoint specifications
- PlantUML diagrams:
  - Use Case Diagram
  - Class Diagram (Domain Model)
  - Component Diagram
  - Entity-Relationship Diagram
  - Sequence Diagrams
  - Activity Diagrams
  - State Diagrams

**Files for Next Stages:**
- `for_code_generation.json` - Entities, APIs, components
- `for_testing.json` - Acceptance criteria, verification methods

## DESMET Evaluation Points

Each stage provides evaluation opportunities for agentic platforms:

### Requirements Stage
- Quality of extracted requirements
- Accuracy of use case identification
- Completeness of entity design
- Correctness of PlantUML diagrams

### Code Generation Stage
- Code correctness and functionality
- Adherence to specifications
- Code quality and style
- Project structure organization

### Testing Stage
- Test coverage
- Test correctness
- Edge case handling
- Test maintainability

### Build & Deploy Stage
- Configuration correctness
- Best practices adherence
- Security considerations
- Scalability support
